# Orchestration Logic (완전 자동화 및 자가 진화)

본 문서는 AI 에이전트가 단 한 번의 프롬프트로 전 과정을 주파하고, 시스템을 스스로 진화시키기 위한 워크플로우를 정의합니다.

## 1. Master Prompt Flow (6단계 자동화 파이프라인)
에이전트는 새로운 태스크를 받으면 다음 단계를 멈춤 없이 완수합니다.

1. **Pre-flight Analysis (사전 분석):** 코드를 수정하기 전, 수정 대상 파일과 예상되는 사이드 이펙트를 먼저 도출하여 스스로 검증합니다.
2. **Virtual Sandbox (Git 격리):** 무조건 터미널에서 `git checkout -b worktree/[task-name]` 명령어로 본류(Main) 코드를 보호합니다.
3. **Develop & Debug:** 코드 작성 및 로그/캡처 기반의 무한 셀프 디버깅 루프를 가동합니다.
4. **Verify (기계적 검증):** `npm run harness:verify`를 돌려 정합성을 증명합니다. (작업 규모에 따라 부분 검증 가능)
5. **Senior Code Review (자체 리뷰):** `04_feedback_loops.md`의 6대 잣대와 **'악마의 대변인(Devil's Advocate)'** 관점에서 리뷰 아티팩트를 작성합니다.
6. **Submit PR (검토 요청):** 리뷰 통과 시 작업 내용을 커밋 및 푸시합니다. 커밋 메시지는 **Conventional Commits** 규격을 준수하며, **시니어 리뷰어의 관점에서 작성된 '자동 코드 리뷰 댓글'이 포함된 PR 형태의 작업 보고서**를 제출하여 사용자의 최종 검토를 대기합니다.

## 2. 자가 진화 규칙 (Self-Evolution)
에이전트는 단순한 코드 작성을 넘어 하네스 시스템 전체의 지능을 높일 의무가 있습니다.
- 작업 중 발견된 프로젝트 특유의 에지 케이스나, 반복되는 실수를 방어하기 위한 새로운 규칙은 즉시 관련 하네스 문서(`01`~`05`)에 업데이트합니다.
- **"하네스 자체를 하나의 살아있는 제품으로 취급하고 지속적으로 개선하세요."**

## 3. 효율적 맥락 수집 및 소통
- FSD 계층 구조에 따라 관련된 Slice 폴더와 `index.ts`를 우선 조회하여 토큰 효율을 극대화합니다.
- 에이전트 스스로 해결할 수 없는 치명적인 블로커(Blocker) 상황에서만 예외적으로 사용자에게 질문합니다.
